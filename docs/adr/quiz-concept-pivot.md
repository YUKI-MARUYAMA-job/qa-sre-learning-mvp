# クイズ問題コンセプトの転換

[docs/adr/quiz-concept-pivot.md](/docs/adr/quiz-concept-pivot.md)

## コンセプト転換に至る経緯

当初のクイズ問題は、外部技術学習や外部機関試験対策に寄った構成としていました。しかし、公開ポートフォリオとして提示するうえでは、以下の懸念がありました。

- 外部公式問題や過去問の再現と誤認されるリスク
- 第三者教材への依存
- 問題の根拠をrepo内部で追跡しづらいこと
- 本ポートフォリオ成果物の説明力に直結しにくいこと

## 方針

```text
クイズ問題のコンセプトを、外部機関試験対策から、本リポジトリの技術スタック、品質ゲート、データ検証、E2E、デプロイ構成を理解するための内製クイズへ転換する。
```

## 結果

この転換により、以下を実現しました。

- 外部公式問題再現リスクの低減
- repo内部source pathによる根拠追跡
- 本ポートフォリオ成果物との関連性向上
- QA/SRE観点での説明可能性向上
- schema / taxonomy / policy validationの一貫性向上
- public quiz dataとquiz quality reportの生成物同期
- 面接・コードレビュー時に説明しやすい内製教材としてのクイズアプリ作成
